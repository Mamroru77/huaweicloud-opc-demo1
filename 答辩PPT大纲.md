## 第2页：背景与痛点 (60秒)

**核心数据**：
- 一篇医学论文 → 临床诊疗方案，传统路径需要 **3-5 年**
- 中国肝癌年新发病例 **40 万例**，术后 5 年复发率 >50%
- 术中精准识别切缘是降低复发的关键，但现有技术不足

**痛点三角**：
```
      临床需求
       /     \
      /       \
  技术壁垒 —— 资金缺口
```
- 临床端：缺少快速技术转化通道
- 技术端：论文到方案需要跨学科专家团队数月工作
- 资金端：创新器械融资缺乏专业计划书支撑

**一句话定位**：
> 用 AI Agent 将"论文→临床方案"的转化周期从**数年缩短到数十秒**

## 第4页：Agent 1 — 技术理解 (60秒)

**角色**：资深生物医学工程专家

**能力**：从论文中自动提取 10+ 项关键技术参数

| 参数类别 | 提取内容 |
|----------|----------|
| 光学参数 | 激发波长、发射波长、组织穿透深度 |
| 药代动力学 | 代谢半衰期、代谢途径、排泄方式 |
| 毒理学 | LD50、肝毒性、肾毒性、遗传毒性 |
| 靶向性 | 靶向组织、靶向受体、特异性数据 |

| 2 | 图迈 Toumai | 9/10 | ~85台 |
| 3 | 康多 KD-SR | 8/10 | ~60台 |

## 第6页：Agent 3 — 执行输出 (90秒)

**角色**：临床研究设计专家 + 医疗投融资顾问

**输出文档一：《临床试验设计方案草案 (Phase I)》**

包含 8 大章节：
1. 试验标题与 NMPA/ClinicalTrials 注册框架
2. 主要/次要终点指标（安全 + 有效）
3. 3+3 剂量递增设计
4. 受试者入排标准（入选 8 条 + 排除 8 条）
5. 样本量估算依据
6. 给药方案与剂量设计
7. 安全性监测计划（含 DLT 定义）
8. 数据管理与统计分析计划

**输出文档二：《创新医疗器械融资计划书》**

包含 8 大章节：
1. 项目概述与核心价值主张
2. 市场分析（短期 SAM 8.75亿，中期 >25亿）
3. 技术壁垒与竞争对比矩阵
4. 团队框架
5. 产品管线规划（HCC → 胰腺癌 → 结直肠癌）
6. 融资金额与资金用途（A轮 1.5亿）
7. 3-5年财务预测
8. 退出策略

## 第7页：技术架构与平台 (60秒)

**技术栈**：
| 层级 | 技术选型 |
|------|----------|
| 前端 | Streamlit（轻量Web框架） |
| 后端 | Python 3.14 |
| AI 引擎 | DeepSeek-Chat（兼容 OpenAI 接口，可替换） |
| LLM 调用 | 统一 API 客户端，支持任意兼容模型 |
| PDF 解析 | PyMuPDF |
| 文档生成 | python-docx + Markdown |

**部署平台**：华为云码道（CodeArts）代码智能体

**项目结构**：
```
agents/        # 3个独立Agent模块
core/          # LLM客户端 + Prompt模板 + 编排器
input/         # 论文解析
web/           # Streamlit界面
output/        # 生成文档输出
```

## 第8页：核心创新点 / 夺冠亮点 (90秒)

> **对应赛事文档："结合了 AI4S 趋势，技术门槛极高，社会价值大"**

### 亮点一：AI4S（AI for Science）
- 不是简单地"让 AI 写代码"，而是让 AI **参与科学研究流程**
- 将 NLP + 领域专家知识结合，实现论文 → 临床方案的自动转化
- 每个 Agent 的 Prompt 都经过生物医学领域专家级设计

### 亮点二：高技术门槛
- 三道知识壁垒合一：生物医学 + 医疗器械 + 临床法规
- 三个 Agent 协作而非单一模型调用，体现真正的多智能体架构
- 输出符合 NMPA、ICH-GCP 等专业标准

### 亮点三：全链路可落地
- 输入：一篇论文 → 输出：可直接提交的注册文档和融资材料
- 完美契合"一人公司"理念：一个人 + AI = 一个转化医学团队
- 已在华为云码道平台完整部署运行
